# 用户级 CLAUDE.md 示例

这是一个用户级 CLAUDE.md 文件的示例。放置在 `~/.claude/CLAUDE.md`。

用户级配置全局适用于所有项目。用于：

* 个人编码偏好
* 您始终希望执行的通用规则
* 指向模块化规则的链接

***

## 核心理念

您是 Claude Code。我使用专门的智能体和技能处理复杂任务。

**关键原则：**

1. **智能体优先**：将复杂工作委托给专门的智能体
2. **并行执行**：尽可能使用 Task 工具配合多个智能体
3. **先计划后执行**：对复杂操作使用计划模式
4. **测试驱动**：先写测试再实现
5. **安全第一**：绝不妥协安全性

## 个人偏好

### 隐私

* 始终对日志进行脱敏；绝不粘贴密钥（API 密钥/令牌/密码/JWT）
* 分享前审查输出 - 移除任何敏感数据

### 代码风格

* 代码、注释或文档中不使用表情符号
* 偏好不可变性 - 永不改变对象或数组
* 多个小文件优于少数大文件
* 典型 200-400 行，每文件最多 800 行

### Git

* 约定式提交：`feat:`、`fix:`、`refactor:`、`docs:`、`test:`
* 提交前始终在本地测试
* 小型、专注的提交

### 测试

* TDD：先写测试
* 最低 80% 覆盖率
* 关键流程进行单元 + 集成 + E2E 测试

### 知识捕获

* 个人调试笔记、偏好和临时上下文 → 自动记忆
* 团队/项目知识（架构决策、API 变更、实现操作手册） → 遵循项目现有的文档结构
* 如果当前任务已经生成了相关文档、注释或示例，请勿在其他地方重复相同知识
* 如果没有明显的项目文档位置，请在创建新的顶级文档前询问
