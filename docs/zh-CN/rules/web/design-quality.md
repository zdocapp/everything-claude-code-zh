# Web 设计质量标准

## 反模板政策

不要交付看起来像通用模板的 UI。前端输出应看起来是经过深思熟虑、有明确主张且专属于产品的。

### 禁止的模式

* 默认的卡片网格，具有均匀的间距且没有层级结构
* 千篇一律的英雄区域，包含居中的标题、渐变斑点图和通用的行动号召按钮
* 将未经修改的库默认样式当作成品设计交付
* 扁平布局，没有层次、深度或动效
* 每个组件都使用统一的圆角、间距和阴影
* 安全的灰白配色方案，仅使用一种装饰性强调色
* 按部就班的仪表板布局（侧边栏 + 卡片 + 图表），缺乏独特视角
* 在没有明确理由的情况下使用默认字体栈

### 必备品质

每个有意义的前端界面都应至少体现以下四项品质：

1. 通过尺度对比展现清晰的层级结构
2. 间距具有有意识的节奏感，而非处处使用统一的填充
3. 通过重叠、阴影、表面或动效来体现深度或层次感
4. 具有特色字体和真实配对策略的排版
5. 语义化地使用颜色，而非仅用于装饰
6. 经过精心设计的悬停、聚焦和激活状态
7. 在适当情况下，采用打破常规网格的编辑性或便当式构图
8. 符合视觉方向时，运用纹理、颗粒感或氛围感
9. 动效用于阐明流程，而非分散注意力
10. 将数据可视化视为设计系统的一部分，而非事后补充

## 编写前端代码之前

1. 选择一个具体的风格方向。避免使用"简洁极简"等模糊的默认选项。
2. 有意识地定义调色板。
3. 审慎地选择字体。
4. 至少收集一小部分真实的参考案例。
5. 在相关处运用 ECC 设计/前端技能。

## 值得考虑的风格方向

* 编辑/杂志风格
* 新粗野主义
* 具有真实深度的玻璃态拟物化
* 具有严谨对比度的暗黑奢华或明亮奢华风格
* 便当式布局
* 滚动叙事
* 3D 集成
* 瑞士/国际主义风格
* 复古未来主义

不要自动默认为深色模式。选择产品真正需要的视觉方向。

## 组件检查清单

* \[ ] 它是否避免了看起来像默认的 Tailwind 或 shadcn 模板？
* \[ ] 它是否具有经过设计的悬停/聚焦/激活状态？
* \[ ] 它是否使用了层级结构而非统一的强调方式？
* \[ ] 它看起来是否像真实产品截图那样可信？
* \[ ] 如果它支持多种主题，其浅色和深色主题是否都感觉是经过精心设计的？
