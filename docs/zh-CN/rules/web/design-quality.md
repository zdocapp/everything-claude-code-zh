# Web 设计质量标准

## 反模板政策

不要交付看起来像通用模板的 UI。前端输出应看起来是经过深思熟虑、有明确主张且专属于产品的。

### 禁止的模式

* 默认的卡片网格，间距统一且没有层级结构
* 千篇一律的英雄区域，包含居中的标题、渐变斑点图和通用的行动号召按钮
* 未经修改的库默认样式，被当作已完成的设计
* 扁平布局，没有层次、深度或动效
* 每个组件都使用统一的圆角、间距和阴影
* 安全的灰白配色，仅使用一种装饰性强调色
* 按数字拼凑的仪表板布局（侧边栏 + 卡片 + 图表），没有明确的观点
* 在没有深思熟虑的情况下使用默认字体栈

### 必备品质

每个有意义的前端界面至少应体现以下品质中的四项：

1. 通过尺度对比建立清晰的层级
2. 间距具有有意的节奏感，而非处处使用统一的填充
3. 通过重叠、阴影、表面或动效来体现深度或层次感
4. 具有特色且真正经过搭配策略考虑的排版
5. 色彩用于语义表达，而不仅仅是装饰
6. 悬停、聚焦和激活状态经过精心设计
7. 在适当的地方采用打破常规网格的编辑性或便当式构图
8. 当符合视觉方向时，运用纹理、颗粒感或氛围感
9. 动效用于阐明流程，而非分散注意力
10. 将数据可视化视为设计系统的一部分，而非事后补充

## 在编写前端代码之前

1. 选择一个具体的风格方向。避免使用诸如“简洁极简”之类的模糊默认选项。
2. 有意识地定义调色板。
3. 审慎地选择排版。
4. 至少收集一小组真实的参考案例。
5. 在相关处运用 ECC 设计/前端技能。

## 值得考虑的风格方向

* 编辑/杂志风格
* 新粗野主义
* 具有真实深度的玻璃态拟物化
* 暗黑奢华或明亮奢华，并带有克制的对比度
* 便当式布局
* 滚动叙事
* 3D 集成
* 瑞士/国际主义风格
* 复古未来主义

不要自动默认为深色模式。选择产品真正需要的视觉方向。

## 组件检查清单

* \[ ] 它是否避免了看起来像默认的 Tailwind 或 shadcn 模板？
* \[ ] 它是否具有经过设计的悬停/聚焦/激活状态？
* \[ ] 它是否使用层级结构而非统一的强调方式？
* \[ ] 它看起来是否像真实产品截图那样可信？
* \[ ] 如果它支持多种主题，那么浅色和深色主题是否都感觉是经过精心设计的？
