# PHP 模式

> 本文档扩展了 [common/patterns.md](../common/patterns.md) 的内容，并添加了 PHP 相关的具体内容。

## 精简控制器，明确服务

* 保持控制器专注于传输层：认证、验证、序列化、状态码。
* 将业务规则移至应用/领域服务中，这些服务易于测试，无需 HTTP 引导。

## DTO 与值对象

* 对于请求、命令和外部 API 负载，使用 DTO 替代结构复杂的关联数组。
* 对于金额、标识符、日期范围和其他受约束的概念，使用值对象。

## 依赖注入

* 依赖接口或窄化的服务契约，而非框架全局变量。
* 通过构造函数传递协作者，使得服务无需服务定位器查找即可测试。

## 边界

* 当模型层所做的不仅仅是持久化时，应将 ORM 模型与领域决策隔离。
* 将第三方 SDK 包装在小型适配器之后，使代码库的其余部分依赖于你的契约，而非他们的。
