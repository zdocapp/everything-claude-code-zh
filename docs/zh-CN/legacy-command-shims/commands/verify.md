# 验证命令（旧版兼容层）

仅当您仍调用 `/verify` 时使用此命令。维护的工作流位于 `skills/verification-loop/SKILL.md`。

## 规范入口

* 优先直接使用 `verification-loop` 技能。
* 保留此文件仅作为兼容性入口点。

## 参数

`$ARGUMENTS`

## 委托

应用 `verification-loop` 技能。

* 根据用户请求的模式选择适当的验证深度。
* 按当前仓库的正确顺序执行构建、类型检查、代码检查、测试、安全/日志检查以及差异审查。
* 仅报告结论和阻塞项，无需在此维护第二份验证清单。
