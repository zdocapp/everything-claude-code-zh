从 $ARGUMENTS 解析以下内容：

1. `brief` — 用户对要创建的设计的描述
2. `--max-iterations N` — (可选，默认 10) 最大设计-评估循环次数
3. `--pass-threshold N` — (可选，默认 7.5) 通过所需的加权分数（设计模式默认值更高）

## GAN 风格设计工具

一个专注于前端设计质量的双代理循环（生成器 + 评估器）。没有规划器——简报本身就是规范。

这是 Anthropic 用于其前端设计实验的相同模式，他们在此模式下看到了创造性的突破，例如使用 CSS 透视和门道导航的 3D 荷兰艺术博物馆。

### 设置

1. 创建 `gan-harness/` 目录
2. 直接将简报写入 `gan-harness/spec.md`
3. 编写一个侧重于设计的 `gan-harness/eval-rubric.md`，并在设计质量和原创性上赋予额外权重

### 设计专用评估标准

```markdown
### 设计质量（权重：0.35）
### 原创性（权重：0.30）
### 工艺（权重：0.25）
### 功能性（权重：0.10）
```

注意：原创性权重更高（0.30 对比 0.20），以推动创造性突破。功能性权重较低，因为设计模式侧重于视觉质量。

### 循环

与 `/project:gan-build` 第 2 阶段相同，但：

* 跳过规划器
* 使用侧重于设计的评估标准
* 生成器提示强调视觉质量而非功能完整性
* 评估器提示强调“这个设计能赢得设计奖项吗？”而非“所有功能都有效吗？”

### 与 gan-build 的关键区别

生成器被告知：“你的首要目标是视觉卓越。一个惊艳的半成品应用胜过功能齐全但丑陋的应用。推动创造性飞跃——不寻常的布局、自定义动画、独特的色彩运用。”
