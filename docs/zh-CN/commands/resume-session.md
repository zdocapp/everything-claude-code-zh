# 恢复会话命令

加载上次保存的会话状态，并在开始任何工作前完全熟悉上下文。
此命令是 `/save-session` 的对应命令。

## 何时使用

* 开始新会话以继续前一天的工作时
* 因上下文限制而启动全新会话后
* 从其他来源交接会话文件时（只需提供文件路径）
* 任何拥有会话文件并希望 Claude 在继续前完全吸收其内容时

## 用法

```
/resume-session                                                      # 加载 ~/.claude/session-data/ 中最新的文件
/resume-session 2024-01-15                                           # 加载该日期最新的会话
/resume-session ~/.claude/session-data/2024-01-15-abc123de-session.tmp  # 加载一个当前短ID会话文件
/resume-session ~/.claude/sessions/2024-01-15-session.tmp               # 加载一个特定的旧格式文件
```

## 流程

### 步骤 1：查找会话文件

如果未提供参数：

1. 检查 `~/.claude/session-data/`
2. 选择最近修改的 `*-session.tmp` 文件
3. 如果文件夹不存在或没有匹配的文件，告知用户：
   ```
   在 ~/.claude/session-data/ 中未找到会话文件。
   请在会话结束时运行 /save-session 来创建一个。
   ```
   然后停止。

如果提供了参数：

* 如果看起来像日期（`YYYY-MM-DD`），首先搜索 `~/.claude/session-data/`，然后是旧的 `~/.claude/sessions/`，查找匹配 `YYYY-MM-DD-session.tmp`（旧格式）或 `YYYY-MM-DD-<shortid>-session.tmp`（当前格式）的文件，并加载该日期最近修改的变体
* 如果看起来像文件路径，直接读取该文件
* 如果未找到，清晰报告并停止

### 步骤 2：读取整个会话文件

读取完整文件。暂时不要总结。

### 步骤 3：确认理解

使用以下确切格式的结构化简报进行回复：

```
会话已加载：[文件的实际解析路径]
════════════════════════════════════════════════

项目：[项目名称 / 文件主题]

我们正在构建什么：
[用你自己的话总结2-3句话]

当前状态：
通过：已确认 [数量] 项工作正常
 进行中：[列出正在处理的文件]
 未开始：[列出计划但未开始的文件]

不应重试的内容：
[列出每个失败的方法及其原因——这至关重要]

待解决问题 / 阻碍：
[列出任何阻碍或未解答的问题]

下一步：
[文件中定义的明确下一步]
[如果未定义："未定义下一步——建议在开始前共同回顾'尚未尝试的内容'"]

════════════════════════════════════════════════
准备继续。您想做什么？
```

### 步骤 4：等待用户

不要自动开始工作。不要触碰任何文件。等待用户指示下一步操作。

如果会话文件中明确定义了下一步，并且用户说"继续"或"是"或类似的话——则执行该确切步骤。

如果未定义下一步——询问用户从哪里开始，并可选地根据"尚未尝试的内容"部分建议一个方法。

***

## 边缘情况

**同一日期有多个会话**（`2024-01-15-session.tmp`, `2024-01-15-abc123de-session.tmp`）：
加载该日期最近修改的匹配文件，无论其使用旧的无 ID 格式还是当前的短 ID 格式。

**会话文件引用了不再存在的文件：**
在简报中注明——"警告：会话中引用了 `path/to/file.ts`，但在磁盘上未找到。"

**会话文件来自 7 天前：**
注明时间间隔——"警告：此会话来自 N 天前（阈值：7 天）。情况可能已发生变化。"——然后正常进行。

**用户直接提供了文件路径（例如，从队友处转发）：**
读取它并遵循相同的简报流程——无论来源如何，格式都是相同的。

**会话文件为空或格式错误：**
报告："找到会话文件，但似乎为空或无法读取。您可能需要使用 /save-session 创建一个新的。"

## 注意事项

* 加载时切勿修改会话文件——它是只读的历史记录
* 简报格式是固定的——即使某些部分为空，也不要跳过
* "不应重试的内容"必须始终显示，即使它只是说"无"——这一点非常重要，不容错过
* 恢复后，用户可能希望在新的会话结束时再次运行 `/save-session` 以创建新的带日期文件
