---

一种结构化的功能开发工作流程，强调在编写新代码之前先理解现有代码。

## 阶段

### 1. 探索

* 仔细阅读功能需求
* 识别需求、约束条件和验收标准
* 如果需求不明确，提出澄清性问题

### 2. 代码库探索

* 使用 `code-explorer` 分析相关的现有代码
* 追踪执行路径和架构层级
* 理解集成点和约定

### 3. 澄清问题

* 呈现探索阶段的发现
* 提出有针对性的设计和边界情况问题
* 在继续之前等待用户回应

### 4. 架构设计

* 使用 `code-architect` 设计功能
* 提供实现蓝图
* 在实施前等待批准

### 5. 实施

* 按照批准的设计实施功能
* 在适当的情况下优先采用测试驱动开发
* 保持提交小而专注

### 6. 质量审查

* 使用 `code-reviewer` 审查实现
* 处理关键和重要问题
* 验证测试覆盖率

### 7. 总结

* 总结构建的内容
* 列出后续事项或限制
* 提供测试说明
