# 验证命令（旧版适配层）

仅当您仍调用 `/verify` 时使用此命令。维护中的工作流位于 `skills/verification-loop/SKILL.md`。

## 规范使用方式

* 首选直接使用 `verification-loop` 技能。
* 仅将此文件保留为兼容性入口点。

## 参数

`$ARGUMENTS`

## 委托执行

应用 `verification-loop` 技能。

* 根据用户请求的模式选择合适的验证深度。
* 为当前仓库按正确顺序运行构建、类型检查、代码检查、测试、安全/日志检查以及差异审查。
* 仅报告结果和阻塞项，而非在此维护第二份验证清单。
