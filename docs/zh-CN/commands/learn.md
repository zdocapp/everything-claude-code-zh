# /learn - 提取可复用模式

分析当前会话，提取值得保存为技能的任何模式。

## 触发条件

在会话期间的任何时刻，当你解决了一个非平凡问题时，运行 `/learn`。

## 提取内容

寻找：

1. **错误解决模式**
   * 发生了什么错误？
   * 根本原因是什么？
   * 如何修复的？
   * 这对于类似错误是否可复用？

2. **调试技术**
   * 非显而易见的调试步骤
   * 有效的工具组合
   * 诊断模式

3. **变通方案**
   * 库的怪癖
   * API 限制
   * 版本特定的修复

4. **项目特定模式**
   * 发现的代码库约定
   * 做出的架构决策
   * 集成模式

## 输出格式

在 `~/.claude/skills/learned/[pattern-name].md` 处创建一个技能文件：

```markdown
# [描述性模式名称]

**提取日期：** [日期]
**适用场景：** [简要描述适用情况]

## 问题描述
[该模式所解决的具体问题]

## 解决方案
[模式/技术/变通方法]

## 示例
[适用时提供代码示例]

## 使用时机
[触发条件——何时应启用此技能]
```

## 流程

1. 回顾会话以寻找可提取的模式
2. 识别最有价值/最可复用的见解
3. 起草技能文件
4. 在保存前请用户确认
5. 保存到 `~/.claude/skills/learned/`

## 注意事项

* 不要提取琐碎的修复（拼写错误、简单的语法错误）
* 不要提取一次性问题（特定的 API 中断等）
* 专注于能节省未来会话时间的模式
* 保持技能聚焦——每个技能一个模式
