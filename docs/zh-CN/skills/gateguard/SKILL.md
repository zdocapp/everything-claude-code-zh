# GateGuard —— 强制事实核查的预操作门控

一个 PreToolUse 钩子，强制 Claude 在编辑前进行调查。它不依赖自我评估（“你确定吗？”），而是要求具体事实。调查行为本身会创造自我评估从未产生的认知。

## 何时激活

* 处理任何文件编辑会影响多个模块的代码库时
* 处理具有特定模式或日期格式的数据文件的项目时
* 在 AI 生成的代码必须匹配现有模式的团队中
* 在任何 Claude 倾向于猜测而非调查的工作流程中

## 核心理念

LLM 的自我评估无效。问“你是否违反了任何政策？”，答案总是“没有”。这已通过实验验证。

但问“列出导入此模块的每个文件”会强制 LLM 运行 Grep 和 Read。调查本身会创造改变输出的上下文。

**三阶段门控：**

```
1. DENY  — 阻止首次编辑/写入/Bash尝试
2. FORCE — 明确告知模型需要收集哪些事实
3. ALLOW — 在事实呈现后允许重试
```

没有其他工具能同时做到这三点。大多数工具止步于拒绝。

## 证据

两项独立的 A/B 测试，相同的智能体，相同的任务：

| 任务 | 启用门控 | 未启用门控 | 差距 |
| --- | --- | --- | --- |
| 分析模块 | 8.0/10 | 6.5/10 | +1.5 |
| Webhook 验证器 | 10.0/10 | 7.0/10 | +3.0 |
| **平均** | **9.0** | **6.75** | **+2.25** |

两个智能体生成的代码都能运行并通过测试。差异在于设计的深度。

## 门控类型

### 编辑 / 多文件编辑门控（每个文件的首次编辑）

多文件编辑的处理方式相同 —— 批次中的每个文件都单独进行门控。

```
在编辑 {file_path} 之前，请提供以下信息：

1. 列出所有导入/引用此文件的文件（使用 Grep）
2. 列出受此更改影响的公共函数/类
3. 如果此文件读取/写入数据文件，请显示字段名称、结构以及日期格式（使用脱敏或合成数据，而非原始生产数据）
4. 逐字引用用户当前的指令
```

### 写入门控（首次创建新文件）

```
在创建 {file_path} 之前，请呈现以下事实：

1. 命名将调用此新文件的文件及行号
2. 确认没有现有文件服务于相同目的（使用 Glob）
3. 如果此文件读取/写入数据文件，请展示字段名称、结构以及日期格式（使用脱敏或合成值，而非原始生产数据）
4. 逐字引用用户当前的指令
```

### 破坏性 Bash 命令门控（每次破坏性命令）

在以下命令时触发：`rm -rf`, `git reset --hard`, `git push --force`, `drop table` 等。

```
1. 列出此命令将修改或删除的所有文件/数据
2. 编写一行回滚步骤
3. 逐字引用用户当前的指令
```

### 常规 Bash 命令门控（每次会话一次）

```
1. 当前用户请求的一句话概括
2. 此特定命令验证或生成的内容
```

## 快速开始

### 选项 A：使用 ECC 钩子（零安装）

插件中包含位于 `scripts/hooks/gateguard-fact-force.js` 的钩子。通过 hooks.json 启用它。

如果 GateGuard 阻止了设置或修复工作，请使用
`ECC_GATEGUARD=off` 启动会话。如需钩子级别的控制，请继续使用
`ECC_DISABLED_HOOKS` 并指定 GateGuard 钩子 ID。

### 选项 B：包含配置的完整包

```bash
pip install gateguard-ai
gateguard init
```

这会添加 `.gateguard.yml` 用于每个项目的配置（自定义消息、忽略路径、门控开关）。

## 反模式

* **不要用自我评估替代。** “你确定吗？”总是得到“是的”。这已通过实验验证。
* **不要跳过数据模式检查。** 两项 A/B 测试中的智能体都假设日期是 ISO-8601 格式，而实际数据使用 `%Y/%m/%d %H:%M`。检查数据结构（使用脱敏值）可以防止整个这类错误。
* **不要门控每一个 Bash 命令。** 常规 bash 命令每个会话门控一次。破坏性 bash 命令每次都会门控。这种平衡避免了速度下降，同时捕捉了真正的风险。

## 最佳实践

* 让门控自然触发。不要试图预先回答门控问题 —— 调查行为本身才是提高质量的关键。
* 为你的领域自定义门控消息。如果你的项目有特定的约定，请将它们添加到门控提示中。
* 使用 `.gateguard.yml` 来忽略路径，例如 `.venv/`, `node_modules/`, `.git/`。

## 相关技能

* `safety-guard` —— 运行时安全检查（互补，不重叠）
* `code-reviewer` —— 编辑后审查（GateGuard 是编辑前调查）
