# 前端设计

当任务不仅仅是“让它能用”，而是“让它看起来经过设计”时，使用此技能。

此技能适用于产品页面、仪表板、应用外壳、组件或视觉系统，这些需要明确的观点，而非通用的、看起来像AI生成的UI。

## 何时使用

* 从零开始构建着陆页、仪表板或应用界面
* 将平淡的界面升级为有意图且令人难忘的界面
* 将产品概念转化为具体的视觉方向
* 实现一个排版、构图和动效都很重要的前端

## 核心原则

选择一个方向并坚持执行。

安全但平庸的UI通常不如具有一些大胆选择、强烈且连贯的美学。

## 设计工作流程

### 1. 首先框定界面

在编码之前，确定：

* 目的
* 受众
* 情感基调
* 视觉方向
* 用户应该记住的一件事

可能的方向：

* 极致简约
* 编辑风格
* 工业风格
* 奢华风格
* 趣味风格
* 几何风格
* 复古未来主义
* 柔和有机风格
* 极繁主义

不要随意混合方向。选择一个并清晰地执行。

### 2. 构建视觉系统

定义：

* 字体层级
* 颜色变量
* 间距节奏
* 布局逻辑
* 动效规则
* 表面/边框/阴影处理

使用CSS变量或项目的令牌系统，以便界面在增长过程中保持连贯。

### 3. 有意图地构图

优先考虑：

* 当不对称能强化层级时，使用不对称
* 当重叠能创造深度时，使用重叠
* 当强留白能明确焦点时，使用强留白
* 仅当产品受益于密集性时，才使用密集布局

除非明显合适，否则避免默认使用对称的卡片网格。

### 4. 让动效有意义

使用动画来：

* 揭示层级
* 分阶段展示信息
* 强化用户操作
* 创造一两个令人难忘的时刻

不要到处散布通用的微交互。一个精心设计的加载序列通常比二十个随机的悬停效果更有效。

## 强力默认项

### 排版

* 选择有特色的字体
* 在适当的时候，将独特的展示字体与易读的正文字体配对
* 当页面以设计为主导时，避免使用通用的默认字体

### 颜色

* 坚持使用清晰的调色板
* 一个主导色域加上选择性点缀通常比权重均匀的彩虹调色板效果更好
* 避免使用陈词滥调的紫色渐变白色背景，除非产品确实需要它

### 背景

运用氛围：

* 渐变
* 网格
* 纹理
* 细微噪点
* 图案
* 分层透明度

对于面向产品的页面，平坦空旷的背景很少是最佳选择。

### 布局

* 当构图受益时，打破网格
* 有意图地使用对角线、偏移和分组
* 即使布局非常规，也要保持阅读流程的清晰

## 反模式

切勿默认使用：

* 可互换的SaaS英雄区域
* 没有层级的通用卡片堆
* 没有系统的随机强调色
* 感觉像占位符的排版
* 仅仅因为动画容易添加而存在的动效

## 执行规则

* 在现有产品内部工作时，保留已建立的设计系统
* 使技术复杂度与视觉创意相匹配
* 保持可访问性和响应性
* 前端在桌面和移动设备上都应感觉是经过深思熟虑的

## 质量门

交付前检查：

* 界面具有清晰的视觉观点
* 排版和间距感觉是有意图的
* 颜色和动效支持产品，而非随意装饰
* 结果读起来不像通用的AI UI
* 实现是生产级的，而不仅仅是视觉上有趣
