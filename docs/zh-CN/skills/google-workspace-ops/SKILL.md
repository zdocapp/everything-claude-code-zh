# Google Workspace 运维

此技能用于将共享文档、电子表格和演示文稿作为工作系统进行操作，而不仅仅是孤立地编辑单个文件。

## 使用时机

* 用户需要查找文档、表格或演示文稿并就地更新
* 整合存储在 Google 云端硬盘中的计划、跟踪器、笔记或客户列表
* 清理或重构共享电子表格
* 导入、修复或重新格式化 Google 幻灯片演示文稿
* 从文档、表格或幻灯片中生成摘要以辅助决策

## 首选工具界面

使用 Google 云端硬盘作为入口点，然后切换到合适的专业工具：

* Google 文档用于文本密集型文档
* Google 表格用于表格工作、公式和图表
* Google 幻灯片用于演示文稿、导入、模板迁移和清理

不要仅凭文件名猜测结构。先进行检查。

## 工作流程

### 1. 查找资产

从云端硬盘搜索界面开始，定位：

* 确切的文件
* 相关资产
* 可能的重复项
* 最近修改的版本

如果有多个文档看起来相似，请通过标题、所有者、修改时间或文件夹进行确认。

### 2. 编辑前检查

在进行更改之前：

* 总结当前结构
* 识别标签页、标题或幻灯片数量
* 判断任务是局部清理还是结构大调整

选择能够安全完成工作的最小工具。

### 3. 精确编辑

* 对于文档：使用基于索引的编辑，而非模糊重写
* 对于表格：在明确的标签页和范围内操作
* 对于幻灯片：区分内容编辑与视觉清理或模板迁移

如果请求的工作涉及视觉或布局敏感部分，请通过检查和验证进行迭代，而不是一次性进行大规模盲目更新。

### 4. 保持工作系统整洁

当文件是更大工作流程的一部分时，还需揭示：

* 重复的跟踪器
* 过时的演示文稿
* 陈旧文档与规范文档
* 资产是否应归档、合并或重命名

## 输出格式

使用：

```text
资产
- 文件名
- 类型
- 为何这是正确的文件

当前状态
- 结构摘要
- 关键问题或阻碍

行动
- 已进行的编辑或建议的编辑

后续步骤
- 归档 / 合并 / 重复项清理 / 下一个待更新的文件
```

## 良好用例

* “查找当前规划文档并对其进行精简”
* “清理此客户电子表格并向我显示有流失风险的行”
* “将此演示文稿导入幻灯片并使其易于展示”
* “查找当前的跟踪器，而不是过时的重复项”
