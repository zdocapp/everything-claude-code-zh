# 设计系统 — 生成与审计视觉系统

## 何时使用

* 启动需要设计系统的新项目时
* 审计现有代码库的视觉一致性时
* 重新设计之前 — 了解现有状况
* 当界面看起来“不对劲”但无法确定原因时
* 审查涉及样式的拉取请求时

## 工作原理

### 模式 1：生成设计系统

分析您的代码库并生成一个连贯的设计系统：

```
1. 扫描 CSS/Tailwind/styled-components 以查找现有模式
2. 提取：颜色、排版、间距、边框圆角、阴影、断点
3. 研究 3 个竞争对手网站以获取灵感（通过浏览器 MCP）
4. 提出一套设计令牌集（JSON + CSS 自定义属性）
5. 生成 DESIGN.md 文件，阐述每个决策的理由
6. 创建一个交互式 HTML 预览页面（自包含，无依赖）
```

输出：`DESIGN.md` + `design-tokens.json` + `design-preview.html`

### 模式 2：视觉审计

从 10 个维度（每个维度 0-10 分）为您的 UI 评分：

```
1. 色彩一致性 —— 你使用的是调色板还是随机的十六进制值？
2. 排版层级 —— 清晰的 h1 > h2 > h3 > 正文 > 说明文字？
3. 间距节奏 —— 一致的尺度（4px/8px/16px）还是随意的？
4. 组件一致性 —— 相似的元素看起来相似吗？
5. 响应式行为 —— 流畅的还是在断点处断裂的？
6. 深色模式 —— 完整的还是半成品？
7. 动画 —— 有目的的还是多余的？
8. 可访问性 —— 对比度、焦点状态、触摸目标
9. 信息密度 —— 杂乱还是简洁？
10. 细节打磨 —— 悬停状态、过渡效果、加载状态、空状态
```

每个维度都会获得一个分数、具体示例以及包含确切文件:行号的修复建议。

### 模式 3：AI 粗制滥造检测

识别通用的 AI 生成设计模式：

```
- 无处不在的过度渐变
- 默认的紫蓝配色
- 无意义的“玻璃拟态”卡片
- 不该圆角的元素被圆角化
- 滚动时过多的动画效果
- 使用通用渐变背景、居中文字的平庸首屏
- 毫无个性的无衬线字体组合
```

## 示例

**为 SaaS 应用生成：**

```
/design-system generate --style minimal --palette earth-tones
```

**审计现有 UI：**

```
/design-system audit --url http://localhost:3000 --pages / /pricing /docs
```

**检查 AI 粗制滥造：**

```
/design-system slop-check
```
