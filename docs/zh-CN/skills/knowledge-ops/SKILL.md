# 知识操作

管理一个多层知识系统，用于跨多个存储库进行知识的摄取、组织、同步和检索。

优先使用实时工作空间模型：

* 代码工作存在于实际克隆的仓库中
* 活跃的执行上下文存在于 GitHub、Linear 和仓库本地的工作上下文文件中
* 更广泛面向人类的笔记可以存在于非仓库上下文/归档文件夹中
* 持久的跨机器记忆属于知识库，而非影子仓库工作空间

## 何时激活

* 用户希望将信息保存到其知识库
* 将文档、对话或数据摄取到结构化存储中
* 跨系统同步知识（本地文件、MCP 内存、Supabase、Git 仓库）
* 对现有知识进行去重或组织
* 用户说“保存到知识库”、“同步知识”、“关于 X 我知道什么”、“摄取这个”、“更新知识库”
* 任何超出简单记忆检索的知识管理任务

## 知识架构

### 第 1 层：活跃执行事实

* **来源：** GitHub 议题、拉取请求、讨论、发布说明、Linear 议题/项目/文档
* **用于：** 工作的当前运行状态
* **规则：** 如果某事影响活跃的工程计划、路线图、部署或发布，优先将其放在这里

### 第 2 层：Claude 代码记忆（快速访问）

* **路径：** `~/.claude/projects/*/memory/`
* **格式：** 带有 frontmatter 的 Markdown 文件
* **类型：** 用户偏好、反馈、项目上下文、参考信息
* **用于：** 跨对话持久存在的快速访问上下文
* **在会话开始时自动加载**

### 第 3 层：MCP 内存服务器（结构化知识图谱）

* **访问：** MCP 内存工具（create\_entities, create\_relations, add\_observations, search\_nodes）
* **用于：** 跨所有存储记忆的语义搜索、关系映射
* **具有可查询图结构的跨会话持久性**

### 第 4 层：知识库仓库 / 持久文档存储

* **用于：** 精选的持久笔记、会话导出、综合研究、操作员记忆、长篇文档
* **规则：** 当内容不属于仓库拥有的代码时，这是跨机器上下文的首选持久存储

### 第 5 层：外部数据存储（Supabase、PostgreSQL 等）

* **用于：** 结构化数据、大型文档存储、全文搜索
* **适用于：** 内存文件无法容纳的文档、需要 SQL 查询的数据

### 第 6 层：本地上下文/归档文件夹

* **用于：** 面向人类的笔记、归档的游戏计划、本地媒体组织、临时的非代码文档
* **规则：** 可用于信息存储，但不是影子代码工作空间
* **不用于：** 活跃的代码更改或应存在于上游的仓库事实

## 摄取工作流

当需要捕获新知识时：

### 1. 分类

这是什么类型的知识？

* 业务决策 -> 记忆文件（项目类型）+ MCP 内存
* 活跃路线图 / 发布 / 实施状态 -> 优先 GitHub + Linear
* 个人偏好 -> 记忆文件（用户/反馈类型）
* 参考信息 -> 记忆文件（参考类型）+ MCP 内存
* 大型文档 -> 外部数据存储 + 记忆中的摘要
* 对话/会话 -> 知识库仓库 + 记忆中的简短摘要

### 2. 去重

检查此知识是否已存在：

* 在记忆文件中搜索现有条目
* 使用相关术语查询 MCP 内存
* 在创建另一个本地笔记之前，检查信息是否已存在于 GitHub 或 Linear 中
* 不要创建重复项。更新现有条目。

### 3. 存储

写入适当的层：

* 始终更新 Claude 代码记忆以快速访问
* 使用 MCP 内存实现语义可搜索性和关系映射
* 当信息更改活跃项目事实时，优先更新 GitHub / Linear
* 对于持久的、长篇的补充内容，提交到知识库仓库

### 4. 索引

更新任何相关的索引或摘要文件。

## 同步操作

### 对话同步

定期将对话历史同步到知识库：

* 来源：Claude 会话文件、Codex 会话、其他代理会话
* 目标：知识库仓库
* 生成会话索引以便快速浏览
* 提交并推送

### 工作空间状态同步

将重要的工作空间配置和脚本镜像到知识库：

* 生成目录映射
* 提交前编辑敏感配置
* 跟踪随时间的变化
* 不要将知识库或归档文件夹视为实时代码工作空间

### GitHub / Linear 同步

当信息影响活跃执行时：

* 更新相关的 GitHub 议题、拉取请求、讨论、发布说明或路线图线程
* 当工作需要持久的规划上下文时，将支持文档附加到 Linear
* 只有在仍有价值的情况下，才在之后镜像一个本地笔记

### 跨源知识同步

将来自多个来源的知识汇集到一处：

* Claude/ChatGPT/Grok 对话导出
* 浏览器书签
* GitHub 活动事件
* 编写状态摘要，提交并推送

## 记忆模式

```
# 短期：当前会话上下文
使用 TodoWrite 进行会话内任务跟踪

# 中期：项目记忆文件
写入 ~/.claude/projects/*/memory/ 以实现跨会话回忆

# 长期：GitHub / Linear / 知识库
将活跃的执行实况放入 GitHub + Linear
将持久化的综合上下文放入知识库仓库

# 语义层：MCP 知识图谱
使用 mcp__memory__create_entities 创建永久性结构化数据
使用 mcp__memory__create_relations 进行关系映射
使用 mcp__memory__add_observations 为已知实体添加新事实
使用 mcp__memory__search_nodes 查找现有知识
```

## 最佳实践

* 保持记忆文件简洁。归档旧数据，而不是让文件无限增长。
* 在所有知识文件中使用 frontmatter（YAML）存储元数据。
* 存储前先去重。先搜索，然后创建或更新。
* 每个事实集优先使用一个规范归属地。避免同一计划在本地笔记、仓库文件和跟踪文档中存在并行副本。
* 在提交到 Git 之前，编辑敏感信息（API 密钥、密码）。
* 对知识文件使用一致的命名约定（小写短横线命名法）。
* 为条目添加主题/类别标签以便于检索。

## 质量门

在完成任何知识操作之前：

* 未创建重复条目
* 敏感数据已从任何 Git 跟踪文件中编辑
* 索引和摘要已更新
* 已为数据类型选择了适当的存储层
* 在相关处添加了交叉引用
