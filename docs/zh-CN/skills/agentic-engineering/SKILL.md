# 智能体工程

将此技能用于工程工作流，其中AI智能体执行大部分实现工作，而人类负责执行质量和风险控制。

## 操作原则

1. 在执行前定义完成标准。
2. 将工作分解为适合智能体处理的单元。
3. 根据任务复杂性路由模型层级。
4. 使用评估和回归检查进行度量。

## 评估优先循环

1. 定义能力评估和回归评估。
2. 运行基线并捕获失败特征。
3. 执行实现。
4. 重新运行评估并比较差异。

## 任务分解

应用15分钟单元规则：

* 每个单元应可独立验证
* 每个单元应具有单一主导风险
* 每个单元应暴露明确的完成条件

## 模型路由

* Haiku：分类、样板转换、局部编辑
* Sonnet：实现和重构
* Opus：架构、根因分析、多文件不变式

## 会话策略

* 对于紧密耦合的单元，继续使用同一会话。
* 在主要阶段转换后，启动新会话。
* 在里程碑完成后进行压缩，而非在主动调试期间。

## AI生成代码的审查重点

优先审查：

* 不变式和边界情况
* 错误边界
* 安全和身份验证假设
* 隐藏的耦合和部署风险

当自动格式化/代码检查工具已强制执行代码风格时，不要在仅涉及风格的分歧上浪费审查周期。

## 成本纪律

按任务跟踪：

* 模型
* 令牌估算
* 重试次数
* 挂钟时间
* 成功/失败

仅当较低层级模型失败且存在明显的推理差距时，才升级模型层级。
