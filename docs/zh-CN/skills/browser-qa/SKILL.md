# 浏览器质量保证 — 自动化视觉测试与交互

## 使用时机

* 将功能部署到预发布/预览环境后
* 需要验证跨页面的用户界面行为时
* 发布前 — 确认布局、表单、交互功能实际有效
* 审查涉及前端代码的拉取请求时
* 无障碍访问审计和响应式测试

## 工作原理

利用浏览器自动化 MCP（claude-in-chrome、Playwright 或 Puppeteer）像真实用户一样与实时页面进行交互。

### 阶段 1：冒烟测试

```
1. 导航至目标 URL
2. 检查控制台错误（过滤噪音：分析工具、第三方）
3. 验证网络请求中无 4xx/5xx 错误
4. 在桌面和移动视口截取首屏截图
5. 检查核心 Web 指标：LCP < 2.5s, CLS < 0.1, INP < 200ms
```

### 阶段 2：交互测试

```
1. 点击所有导航链接 — 验证无死链
2. 使用有效数据提交表单 — 验证成功状态
3. 使用无效数据提交表单 — 验证错误状态
4. 测试认证流程：登录 → 受保护页面 → 登出
5. 测试关键用户旅程（结账、新用户引导、搜索）
```

### 阶段 3：视觉回归测试

```
1. 在 3 个断点（375px、768px、1440px）处截取关键页面截图
2. 与基线截图（如果已存储）进行比较
3. 标记布局偏移 > 5px、缺失元素、溢出
4. 检查深色模式（如果适用）
```

### 阶段 4：无障碍访问

```
1. 在每个页面上运行 axe-core 或等效工具
2. 标记 WCAG AA 违规项（对比度、标签、焦点顺序）
3. 验证键盘导航端到端工作正常
4. 检查屏幕阅读器地标
```

## 输出格式

```markdown
## QA 报告 — [URL] — [时间戳]

### 冒烟测试
- 控制台错误：0 个严重，2 个警告（分析噪音）
- 网络：全部 200/304，无失败
- 核心 Web 指标：LCP 1.2秒 ✓，CLS 0.02 ✓，INP 89毫秒 ✓

### 交互
- [✓] 导航链接：12/12 正常工作
- [✗] 联系表单：缺少针对无效电子邮件的错误状态
- [✓] 认证流程：登录/登出正常工作

### 视觉
- [✗] 英雄区域在 375px 视口下溢出
- [✓] 深色模式：所有页面一致

### 无障碍性
- 2 个 AA 级违规：英雄图像缺少替代文本，页脚链接对比度低

### 结论：附带修复发布（2 个问题，0 个阻塞项）
```

## 集成

可与任何浏览器 MCP 配合使用：

* `mChild__claude-in-chrome__*` 工具（首选 — 使用您实际的 Chrome 浏览器）
* 通过 `mcp__browserbase__*` 使用 Playwright
* 直接使用 Puppeteer 脚本

与 `/canary-watch` 搭配使用，用于部署后监控。
