# 医疗电子病历开发模式

构建电子病历（EMR）和电子健康档案（EHR）系统的模式。优先考虑患者安全、临床准确性和从业者效率。

## 使用场景

* 构建患者就诊工作流（主诉、检查、诊断、处方）
* 实施临床记录（结构化 + 自由文本 + 语音转文本）
* 设计包含药物相互作用检查的处方/用药模块
* 集成临床决策支持系统（CDSS）
* 构建带有参考范围高亮显示的化验结果展示
* 为临床数据实施审计追踪
* 为临床数据录入设计医疗可访问性用户界面

## 工作原理

### 患者安全第一

每个设计决策都必须依据以下原则进行评估：“这可能会伤害患者吗？”

* 药物相互作用**必须**发出警报，而非静默通过
* 异常化验值**必须**进行视觉标记
* 关键生命体征**必须**触发升级工作流
* 没有审计追踪，不得修改临床数据

### 单页就诊流程

临床就诊应在单个页面上垂直流动——无需切换标签页：

```
患者头部（置顶显示 — 始终可见）
├── 人口统计信息、过敏史、当前用药
│
就诊流程（垂直滚动）
├── 1. 主诉（结构化模板 + 自由文本）
├── 2. 现病史
├── 3. 体格检查（按系统分类）
├── 4. 生命体征（自动触发临床评分）
├── 5. 诊断（ICD-10/SNOMED 搜索）
├── 6. 用药（药品数据库 + 相互作用检查）
├── 7. 检查（检验/放射学医嘱）
├── 8. 计划与随访
└── 9. 签名 / 锁定 / 打印
```

### 智能模板系统

```typescript
interface ClinicalTemplate {
  id: string;
  name: string;             // e.g., "Chest Pain"
  chips: string[];          // clickable symptom chips
  requiredFields: string[]; // mandatory data points
  redFlags: string[];       // triggers non-dismissable alert
  icdSuggestions: string[]; // pre-mapped diagnosis codes
}
```

任何模板中的危险信号都必须触发一个可见的、不可关闭的警报——**不是**一个短暂通知。

### 用药安全模式

```
用户选择药物
  → 检查当前用药是否存在相互作用
  → 检查就诊用药是否存在相互作用
  → 检查患者过敏史
  → 根据体重/年龄/肾功能验证剂量
  → 如果存在 CRITICAL 相互作用：完全阻止处方开具
  → 临床医生必须记录覆盖原因才能绕过阻止
  → 如果存在 MAJOR 相互作用：显示警告，需要确认
  → 将所有警报和覆盖原因记录在审计追踪中
```

关键相互作用**默认阻止处方开具**。临床医生必须明确覆盖，并提供记录在审计追踪中的理由。系统绝不允许静默通过关键相互作用。

### 锁定就诊模式

一旦临床就诊被签署：

* 不允许编辑——只能添加附录（一个单独的链接记录）
* 原始记录和附录都显示在患者时间线中
* 审计追踪记录签署者、签署时间以及任何附录记录

### 临床数据的用户界面模式

**生命体征显示：** 当前值附带正常范围高亮（绿色/黄色/红色），与先前值的趋势箭头对比，自动计算的临床评分（NEWS2，qSOFA），内联的升级指导。

**化验结果显示：** 正常范围高亮，与先前值对比，关键值附带不可关闭的警报，采集/分析时间戳，待处理订单附带预期周转时间。

**处方PDF：** 一键生成，包含患者人口统计信息、过敏史、诊断、药物详情（通用名 + 商品名、剂量、给药途径、频率、持续时间）、临床医生签名栏。

### 医疗可访问性

医疗用户界面比典型的网络应用有更严格的要求：

* 4.5:1 最低对比度（WCAG AA）——临床医生在各种光照条件下工作
* 大的触摸目标（最小 44x44 像素）——适用于戴手套/匆忙的交互
* 键盘导航——供快速录入数据的熟练用户使用
* 无纯颜色指示器——始终将颜色与文本/图标配对（色盲临床医生）
* 所有表单字段都有屏幕阅读器标签
* 临床警报不使用自动关闭的短暂通知——临床医生必须主动确认

### 反模式

* 将临床数据存储在浏览器 localStorage 中
* 药物相互作用检查中的静默失败
* 关键临床警报使用可关闭的短暂通知
* 基于标签页的就诊用户界面，割裂了临床工作流
* 允许编辑已签署/锁定的就诊记录
* 显示没有审计追踪的临床数据
* 对临床数据结构使用 `any` 类型

## 示例

### 示例 1：患者就诊流程

```
Doctor opens encounter for Patient #4521
  → 粘性标题显示："Rajesh M, 58M, 过敏史：青霉素，当前用药：二甲双胍 500mg"
  → 主诉：选择"胸痛"模板
    → 点击标签："胸骨后"、"向左臂放射"、"压榨性"
    → 红色标志"压榨性胸骨后胸痛"触发不可关闭的警报
  → 体格检查：心血管系统 — "S1 S2 正常，无杂音"
  → 生命体征：心率 110，血压 90/60，血氧饱和度 94%
    → NEWS2 自动计算：评分 8，风险 高，显示升级警报
  → 诊断：搜索"ACS" → 选择 ICD-10 I21.9
  → 药物：选择阿司匹林 300mg
    → 临床决策支持系统检查与二甲双胍的相互作用：无相互作用
  → 签署病历 → 锁定，此后仅可添加附录
```

### 示例 2：用药安全工作流

```
医生为患者 #4521 开具华法林处方
  → CDSS 检测到：华法林 + 阿司匹林 = 严重相互作用
  → 用户界面：红色不可关闭模态框阻止处方开具
  → 医生点击"说明理由后覆盖"
  → 输入："获益大于风险 — 已监测 INR 方案"
  → 覆盖理由 + 警报存储在审计追踪中
  → 处方在记录覆盖后继续进行
```

### 示例 3：锁定就诊 + 附录

```
Encounter #E-2024-0891 signed by Dr. Shah at 14:30
  → All fields locked — no edit buttons visible
  → "Add Addendum" button available
  → Dr. Shah clicks addendum, adds: "Lab results received — Troponin elevated"
  → New record E-2024-0891-A1 linked to original
  → Timeline shows both: original encounter + addendum with timestamps
```
