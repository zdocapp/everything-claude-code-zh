# Manim 视频

当运动、结构和清晰度比照片级真实感更重要时，使用 Manim 制作技术解说动画。

## 何时启用

* 用户想要一个技术解说动画
* 概念涉及图表、工作流、架构、指标进展或系统图
* 用户想要一个简短的 X 产品或发布解说视频，或用于落地页
* 视觉上应感觉精确，而非通用的电影感

## 工具要求

* `manim` CLI 用于场景渲染
* `ffmpeg` 用于后期处理（如果需要）
* `video-editing` 用于最终合成或润色
* `remotion-video-creation` 当最终成品需要合成 UI、字幕或额外的运动图层时

## 默认输出

* 简短的 16:9 MP4 视频
* 一张缩略图或海报帧
* 故事板加场景规划

## 工作流程

1. 用一句话定义核心视觉主题。
2. 将概念分解为 3 到 6 个场景。
3. 决定每个场景要证明什么。
4. 在编写 Manim 代码之前，先写好场景大纲。
5. 首先渲染最小可行版本。
6. 在渲染成功后，优化排版、间距、色彩和节奏。
7. 仅在能增加价值时，才移交到更广泛的视频处理栈。

## 场景规划规则

* 每个场景应证明一件事
* 避免图表过于拥挤
* 倾向于渐进式揭示，而非满屏杂乱
* 使用运动来解释状态变化，而不仅仅是为了让屏幕保持忙碌
* 标题卡应简短且富有意义

## 网络图默认设置

对于社交图和网络优化解说：

* 在展示优化后的图之前，先展示当前图
* 区分低信号关注的杂乱连接与高信号的桥梁连接
* 高亮显示热路径节点和目标集群
* 如果有用，可以添加一个最终场景，展示影响该技能形成的自我提升脉络

## 渲染约定

* 默认使用 16:9 横屏，除非用户要求竖屏
* 从低质量的冒烟测试渲染开始
* 仅在构图和时序稳定后，才提升到更高质量
* 导出一张清晰、在社交媒体尺寸下易于阅读的缩略图帧

## 可复用起点

使用 [assets/network\_graph\_scene.py](../../../../skills/manim-video/assets/network_graph_scene.py) 作为网络图解说的起点。

示例冒烟测试：

```bash
manim -ql assets/network_graph_scene.py NetworkGraphExplainer
```

## 输出格式

返回：

* 核心视觉主题
* 故事板
* 场景大纲
* 渲染计划
* 任何后续的润色建议

## 相关技能

* `video-editing` 用于最终润色
* `remotion-video-creation` 用于运动密集的后期处理或合成
* `content-engine` 当动画是更广泛发布活动的一部分时
