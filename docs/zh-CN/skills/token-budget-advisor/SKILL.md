# Token 预算顾问 (TBA)

在 Claude 回答**之前**拦截响应流程，为用户提供关于响应深度的选择。

## 何时使用

* 用户想要控制响应的长度或详细程度
* 用户提及 tokens、预算、深度或响应长度
* 用户说"简短版本"、"tldr"、"简要"、"al 25%"、"详尽"等
* 任何用户希望预先选择深度/详细程度的时候

**不要触发**当：用户已在本会话中设置了级别（静默保持），或者答案明显只有一行。

## 工作原理

### 步骤 1 — 估算输入 tokens

使用仓库的规范上下文预算启发式方法，在脑中估算提示的 token 数量。

使用与 [context-budget](../context-budget/SKILL.md) 相同的校准指导：

* 散文：`words × 1.3`
* 代码为主或混合/代码块：`chars / 4`

对于混合内容，使用主要的内容类型并保持估算启发式方法。

### 步骤 2 — 根据复杂度估算响应大小

对提示进行分类，然后应用乘数范围以获得完整的响应窗口：

| 复杂度       | 乘数范围         | 示例提示                                          |
|--------------|------------------|---------------------------------------------------|
| 简单         | 3× – 8×          | "X 是什么？"，是/否，单一事实                     |
| 中等         | 8× – 20×         | "X 如何工作？"                                    |
| 中高         | 10× – 25×        | 带有上下文的代码请求                               |
| 复杂         | 15× – 40×        | 多部分分析、比较、架构                             |
| 创意         | 10× – 30×        | 故事、文章、叙事写作                               |

响应窗口 = `input_tokens × mult_min` 到 `input_tokens × mult_max`（但不要超过模型配置的输出 token 限制）。

### 步骤 3 — 呈现深度选项

在回答**之前**呈现此块，使用实际估算的数字：

```
分析你的提示...

输入: ~[N] 个词元  |  类型: [类型]  |  复杂度: [级别]  |  语言: [语言]

选择你的深度级别:

[1] 基础   (25%)  ->  ~[词元]   仅直接回答，无前言
[2] 中等    (50%)  ->  ~[词元]   回答 + 上下文 + 1 个示例
[3] 详细    (75%)  ->  ~[词元]   完整回答，包含替代方案
[4] 详尽   (100%)  ->  ~[词元]   全部内容，无限制

选择哪个级别？ (1-4 或说 "25% 深度", "50% 深度", "75% 深度", "100% 深度")

精确度: 启发式估计 ~85-90% 准确率 (±15%)。
```

级别 token 估算（在响应窗口内）：

* 25%  → `min + (max - min) × 0.25`
* 50%  → `min + (max - min) × 0.50`
* 75%  → `min + (max - min) × 0.75`
* 100% → `max`

### 步骤 4 — 按所选级别响应

| 级别            | 目标长度          | 包含内容                                          | 省略内容                                          |
|-----------------|-------------------|---------------------------------------------------|---------------------------------------------------|
| 25% 核心        | 最多 2-4 句       | 直接答案，关键结论                                | 上下文、示例、细微差别、替代方案                  |
| 50% 适中        | 1-3 段            | 答案 + 必要上下文 + 1 个示例                      | 深度分析、边缘情况、参考文献                      |
| 75% 详细        | 结构化响应        | 多个示例、优缺点、替代方案                        | 极端边缘情况、详尽的参考文献                      |
| 100% 详尽       | 无限制            | 一切 — 完整分析、所有代码、所有视角               | 无                                                |

## 快捷方式 — 跳过提问

如果用户已经表明了级别，立即按该级别响应，无需询问：

| 用户所说内容                                    | 级别  |
|-------------------------------------------------|-------|
| "1" / "25% 深度" / "简短版本" / "简要回答" / "tldr"  | 25%   |
| "2" / "50% 深度" / "适中深度" / "平衡回答"           | 50%   |
| "3" / "75% 深度" / "详细回答" / "彻底回答"           | 75%   |
| "4" / "100% 深度" / "详尽回答" / "完整深度探讨"       | 100%  |

如果用户在会话早期设置了级别，**静默保持**该级别用于后续响应，除非他们更改。

## 精度说明

此技能使用启发式估算 — 非真实 tokenizer。准确率约 85-90%，方差 ±15%。始终显示免责声明。

## 示例

### 触发

* "先给我简短版本。"
* "你的回答会使用多少 tokens？"
* "以 50% 深度响应。"
* "我想要详尽的答案，而不是摘要。"
* "Dame la version corta y luego la detallada."

### 不触发

* "JWT token 是什么？"
* "结账流程使用支付 token。"
* "这正常吗？"
* "完成重构。"
* 用户在会话中已选择深度后的后续问题
