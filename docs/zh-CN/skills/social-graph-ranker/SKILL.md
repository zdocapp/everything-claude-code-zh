# 社交图谱排序器

用于网络感知型拓展的规范加权图谱排序层。

当用户需要以下功能时使用此技能：

* 根据介绍价值对现有互相关注者或联系人进行排序
* 绘制通往目标列表的温暖路径
* 衡量一阶和二阶联系人之间的桥梁价值
* 决定哪些目标值得通过温暖介绍而非直接冷接触
* 独立于 `lead-intelligence` 或 `connections-optimizer` 理解图谱数学原理

## 何时单独使用此技能

当用户主要需要排序引擎时选择此技能：

* “我网络中谁最适合介绍我？”
* “根据谁能帮我联系到这些人来对我的互相关注者进行排序”
* “将我的图谱与这个理想客户画像进行匹配”
* “展示桥梁计算的数学原理”

当用户真正需要以下功能时，不要单独使用此技能：

* 完整的潜在客户生成和对外序列 -> 使用 `lead-intelligence`
* 网络修剪、重新平衡和扩展 -> 使用 `connections-optimizer`

## 输入

收集或推断：

* 目标人群、公司或理想客户画像定义
* 用户在 X、LinkedIn 或两者上的当前图谱
* 加权优先级，如角色、行业、地理位置和响应度
* 遍历深度和衰减容忍度

## 核心模型

给定：

* `T` = 加权目标集合
* `M` = 你当前的互相关注者 / 直接联系人
* `d(m, t)` = 从互相关注者 `m` 到目标 `t` 的最短跳数距离
* `w(t)` = 来自信号评分的目标权重

基础桥梁分数：

```text
B(m) = Σ_{t ∈ T} w(t) · λ^(d(m,t) - 1)
```

其中：

* `λ` 是衰减因子，通常为 `0.5`
* 直接路径贡献全部价值
* 每增加一跳，贡献减半

二阶扩展：

```text
B_ext(m) = B(m) + α · Σ_{m' ∈ N(m) \\ M} Σ_{t ∈ T} w(t) · λ^(d(m',t))
```

其中：

* `N(m) \\ M` 是互相关注者认识但你并不认识的人群集合
* `α` 对二阶触及范围进行折扣，通常为 `0.3`

响应调整后的最终排序：

```text
R(m) = B_ext(m) · (1 + β · engagement(m))
```

其中：

* `engagement(m)` 是归一化的响应度或关系强度
* `β` 是参与度奖励，通常为 `0.2`

解读：

* 第 1 层：高 `R(m)` 和直接桥梁路径 -> 温暖介绍请求
* 第 2 层：中等 `R(m)` 和一跳桥梁路径 -> 有条件介绍请求
* 第 3 层：低 `R(m)` 或无可行桥梁 -> 直接接触或填补关注缺口

## 评分信号

在图谱遍历之前，根据当前优先级集合的重要事项对目标进行加权：

* 角色或职位匹配度
* 公司或行业契合度
* 当前活跃度和近期性
* 地理相关性
* 影响力或触及范围
* 响应可能性

在图谱遍历之后，根据以下因素对互相关注者进行加权：

* 通往目标集合的加权路径数量
* 这些路径的直接性
* 响应度或先前互动历史
* 进行介绍的情境契合度

## 工作流程

1. 构建加权目标集合。
2. 从 X、LinkedIn 或两者拉取用户的图谱。
3. 计算直接桥梁分数。
4. 为最高价值的互相关注者扩展二阶候选者。
5. 按 `R(m)` 排序。
6. 返回：
   * 最佳温暖介绍请求
   * 有条件桥梁路径
   * 不存在温暖路径的图谱缺口

## 相关技能

* `lead-intelligence` 在更广泛的目标发现和拓展流程中使用此排序模型
* `connections-optimizer` 在决定保留、修剪或添加谁时使用相同的桥梁逻辑
* `brand-voice` 应在起草任何介绍请求或直接接触之前运行
* `x-api` 提供 X 图谱访问和可选执行路径
