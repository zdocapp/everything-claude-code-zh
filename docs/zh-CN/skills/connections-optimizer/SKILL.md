# 连接优化器

重组用户的网络，而非将对外联系视为单向的潜在客户列表。

此技能处理：

* X 关注清理与扩展
* LinkedIn 关注与连接分析
* 优先审查的清理队列
* 添加与关注推荐
* 暖路径识别
* 以用户真实语气生成 Apple Mail、X DM 和 LinkedIn 草稿

## 何时激活

* 用户希望清理其 X 关注列表
* 用户希望重新平衡其关注或保持联系的对象
* 用户提及“清理我的网络”、“我应该取消关注谁”、“我应该关注谁”、“我应该与谁重新联系”
* 外联质量取决于网络结构，而不仅仅是冷列表生成

## 所需输入

收集或推断：

* 当前优先级和活跃工作
* 目标角色、行业、地域或生态系统
* 平台选择：X、LinkedIn 或两者
* 禁止触碰列表
* 模式：`light-pass`、`default` 或 `aggressive`

如果用户未指定模式，则使用 `default`。

## 工具要求

### 首选

* `x-api` 用于 X 图谱检查和近期活动
* `lead-intelligence` 用于目标发现和暖路径排名
* `social-graph-ranker` 当用户希望独立于更广泛的潜在客户工作流对桥梁价值进行评分时
* Exa / 深度研究用于个人和公司信息丰富
* `brand-voice` 在起草外联内容之前

### 备选方案

* 浏览器控制用于 LinkedIn 分析和草稿撰写
* 如果 API 覆盖受限，则使用浏览器控制处理 X
* 当电子邮件是合适渠道时，通过桌面自动化使用 Apple Mail 或 Mail.app 起草邮件

## 安全默认设置

* 默认设置为优先审查，绝不盲目自动清理
* X：仅清理用户关注的账户，绝不清理关注者
* LinkedIn：将移除一度连接视为需手动优先审查
* 不自动发送 DM、邀请或电子邮件
* 在任何应用步骤之前，输出一个排好序的行动计划和草稿

## 平台规则

### X

* 互相关注比单向关注更稳固
* 未回关的账户可以更积极地清理
* 应快速识别出高度不活跃或已消失的账户
* 互动、信号质量和桥梁价值比原始粉丝数更重要

### LinkedIn

* 如果用户确实拥有 LinkedIn API 访问权限，则优先使用 API
* 当缺少 API 访问权限时，浏览器工作流必须可用
* 区分对外关注与已接受的一度连接
* 对外关注可以更自由地清理
* 已接受的一度连接应默认进行审查，而非自动移除

## 模式

### `light-pass`

* 仅清理高置信度、低价值的单向关注
* 将其余账户列出供审查
* 生成一个小的添加/关注列表

### `default`

* 平衡的清理队列
* 平衡的保留列表
* 排好序的添加/关注队列
* 在有用处时起草暖介绍或直接外联内容

### `aggressive`

* 更大的清理队列
* 对陈旧未回关的容忍度更低
* 在应用前仍需经过审查门控

## 评分模型

使用以下积极信号：

* 互惠性
* 近期活动
* 与当前优先级的契合度
* 网络桥梁价值
* 角色相关性
* 真实的互动历史
* 近期存在和响应性

使用以下消极信号：

* 已消失或废弃的账户
* 陈旧单向关注
* 偏离优先级的主题集群
* 低价值噪音
* 重复无回应
* 存在许多更好的替代者时仍未回关

互相关注和真实的暖路径桥梁应比单向关注受到更少的惩罚。

## 工作流程

1. 捕获优先级、禁止触碰约束和所选平台。
2. 拉取当前的关注/连接清单。
3. 为待清理候选者评分并附上明确理由。
4. 为保留候选者评分并附上明确理由。
5. 使用 `lead-intelligence` 加上研究结果对扩展候选者进行排名。
6. 匹配正确的渠道：
   * X DM 用于温暖、快速的社交接触点
   * LinkedIn 消息用于专业图谱邻近关系
   * Apple Mail 草稿用于更高情境的介绍或外联
7. 在起草消息前运行 `brand-voice`。
8. 在任何应用步骤之前返回一个审查包。

## 外联规则

* 默认电子邮件路径是创建 Apple Mail / Mail.app 草稿。
* 不自动发送。
* 根据温暖度、相关性和情境深度选择渠道。
* 当电子邮件或无外联是正确选择时，不要强制使用 DM。
* 草稿应听起来像用户本人，而非自动化的销售文案。

## 相关技能

* `brand-voice` 用于可重用的语音配置文件
* `social-graph-ranker` 用于独立的桥梁评分和暖路径计算
* `lead-intelligence` 用于加权目标和暖路径发现
* `x-api` 用于 X 图谱访问、草稿撰写和可选的应用流程
* `content-engine` 当用户还希望围绕网络变动发布公开启动内容时
